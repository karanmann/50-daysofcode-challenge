Query 'n Scroll

This Pen shows how to query an array of elements in vanilla JS and filter based on id or class, minus an array of exceptions.

This application takes a non-animated series of sections, adds a scroll trigger to elements of a given class or ID, and then add the corresponding animation class on scroll.

Pages that have scroll-trigger_ classes pre-applied will be triggered on scroll via javascript.

.animation_[name] can be directly attached for immediate animation if you give your element a starting animation state, such as opacity: 0;

.delay-[milliseconds] does what is it says and applies a series of inceasingly longer delays to a series of items to stagger transitions

Motivations

  1. Desire to eliminate jQuery for a common task.
  2. Need for dry code that can get children of an element and filter out nodes by id and class, allowing targeted manipulation.
  3. Desire to speed up query and execution

Modifed with SCSS

In this example, the animation trigger class is set on the top level Bootstrap jumbotrons. Each jumbotron's first container holds the contents that I would like to have fade in on scroll. So, in the scss I have target the first container of each jumbotron.

Filtered Nodes

The getElements function accepts the following:

  • type ( a string of "id" or "class")
  • name ( a string of the id name or class name)
  • parent ( an DOM node, such as document.body)
  • exclusions ( an array of strings/classnames to ignore)

scrollFadeElements = getElements("class", "container", document.body, [ "container-hero", "hero" ]);

It returns an array of nodes to act upon.

div#modifiedWithSCSS.jumbotron.jumbotron-fluid, div#motivations.jumbotron.jumbotron-fluid, div#filteredNodes.jumbotron.jumbotron-fluid, div#motivations.jumbotron.jumbotron-fluid]

Issues Encountered

Arrays gotten from the DOM were array-like (DOM token list) and did not share the same methods as traditional array. mynode.classList, for example returned an array that I could not iterate through. mynode.className.split(''), on the other hand, returned an easily iterable array split off from a complete string of all of the element's classes.

Also see Array.from([somenodearray]) to convert a collection of nodes to something iterable using .forEach.
